Platform Architecture

How should the app launcher and child apps relate?

A

Monorepo with Sub-Apps

One Next.js app with /<app> routes. Finance lives at /finance, future apps at /<name>. Shared auth, DB, and UI shell.

Example: app/launcher/page.tsx + app/finance/page.tsx

B

Multi-App Workspace

Parent monorepo (platform/) manages multiple independent deployables. Each app (finance, etc.) is its own package with shared types.

Example: Turborepo with apps/launcher, apps/finance, apps/notes

C

Standalone with Subdomain

Each app is a fully independent deployment. Launcher lives at app.example.com, finance at finance.example.com. SSO shared across apps.

Example: Separate Vercel/Cloudflare deploys with shared auth provider
